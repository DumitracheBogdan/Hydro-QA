## Notes on batch 2 anomaly

Batch 2 shows 58 unresolved / 0 skipped — investigation: worker 2 classified most "no notes + temperature/flushing" cases under `unresolved_other` rather than `skip=true`. This is a worker-policy variance from the other batches (which used skip=true for non-lab jobTypes). Treated as unresolved is conservative and safe, but recommend tightening worker prompts so non-lab jobTypes are consistently marked `skip` rather than `unresolved` to keep the unresolved bucket meaningful.

## Next-run recommendations

1. Add explicit `skip` heuristic to worker prompts: "If jobType is OT / CF / TI / flushing / on-site testing / form, use `skip=true` not `unresolved` regardless of notes presence."

3. Continue MED-confidence slot-allocation review until a booker-encoded itemLocation convention is adopted.
